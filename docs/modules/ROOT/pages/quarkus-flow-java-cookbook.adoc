= Quarkus Flow & CNCF Serverless Workflow Java Cookbook
:page-role: reference
include::includes/attributes.adoc[]


This cookbook provides a comprehensive collection of practical, copy-and-paste examples for building modern workflows using the Quarkus Flow Java Fluent API.

All workflows in Java are defined as CDI beans by extending the `Flow` class and overriding the `descriptor()` method.

[#pausing_execution]
== 1. Pausing Execution (Wait State)
To pause a workflow for a specific duration (ISO 8601), use the `wait` task.

[source,java]
----
@ApplicationScoped
public class WaitWorkflow extends Flow {
    @Override
    public Workflow descriptor() {
        return FuncWorkflowBuilder.workflow("wait-duration-8601")
            .tasks(waitTask("wait30Seconds").duration("PT30S"))
            .build();
    }
}
----

[#scheduled_execution]
== 2. Scheduled Executions (CRON)
Trigger workflows on a schedule by defining a `schedule` within the builder.

[source,java]
----
@ApplicationScoped
public class CronWorkflow extends Flow {
    @Override
    public Workflow descriptor() {
        return FuncWorkflowBuilder.workflow("cron-schedule")
            .schedule(cron("0 0 * * *")) // Runs daily at midnight
            .tasks(
                // Workflow tasks
            )
            .build();
    }
}
----

[#external_api_interaction]
== 3. External API Integrations (HTTP REST)
Use the `call` task with `http` to interact with REST endpoints, managing headers and query parameters dynamically.

[source,java]
----
@ApplicationScoped
public class HttpWorkflow extends Flow {
    @Override
    public Workflow descriptor() {
        return FuncWorkflowBuilder.workflow("http-query-headers")
            .tasks(
                call("searchStarWarsCharacters")
                    .http(Method.GET, "https://swapi.dev/api/people/")
                    .query("search", "${ .searchQuery }")
                    .header("Accept", "application/json")
            )
            .build();
    }
}
----

[#authentication]
== 5. Authentication (OAuth2)
Define authentication mechanisms for secure outbound communication.

[source,java]
----
@ApplicationScoped
public class SecureHttpWorkflow extends Flow {
    @Override
    public Workflow descriptor() {
        return FuncWorkflowBuilder.workflow("oauth2-authentication")
            .tasks(
                call("getPet")
                    .http(Method.GET, "https://petstore.swagger.io/v2/pet/{petId}")
                    .authentication(oauth2()
                        .authority("http://keycloak/realms/fake-authority")
                        .tokenEndpoint("/auth/token")
                        .grant(GrantType.CLIENT_CREDENTIALS)
                        .clientId("workflow-runtime-id")
                        .clientSecret("workflow-runtime-secret")
                    )
            )
            .build();
    }
}

[#events]
== 6. Event Consumption and Production
To integrate the workflow with CloudEvents. Use `listen` to pause for events and `emit` to publish them.

Listening for an Event:
[source,java]
----
@ApplicationScoped
public class ListenWorkflow extends Flow {
    @Override
    public Workflow descriptor() {
        return FuncWorkflowBuilder.workflow("listen-to-one")
            .tasks(
                listen("waitForStartup", to().one(event("race.started.v1"))),
                call("startup").http(Method.POST, "https://api.race.com/start")
            )
            .build();
    }
}
----

Emitting an Event:
[source,java]
----
@ApplicationScoped
public class EmitWorkflow extends Flow {
    @Override
    public Workflow descriptor() {
        return FuncWorkflowBuilder.workflow("emit-event")
            .tasks(
                emitJson("orderPlaced", "com.petstore.order.placed.v1")
                    .source("https://petstore.com")
                    .data(Map.of("id", "${ .orderId }", "status", "placed"))
            )
            .build();
    }
}
----

[#conditional_logic]
== 8. Conditional Logic (`if` and `switch`)
Branch execution based on data state using `ifCondition` or `switchTask`.

[source,java]
----
@ApplicationScoped
public class SwitchWorkflow extends Flow {
    @Override
    public Workflow descriptor() {
        return FuncWorkflowBuilder.workflow("switch-example")
            .tasks(
                switchTask("processOrder")
                    .caseOf(".orderType == 'electronic'", "processElectronic")
                    .caseOf(".orderType == 'physical'", "processPhysical")
                    .defaultCase("handleUnknown"),

                set("processElectronic", Map.of("status", "fulfilled")).then(exit()),
                set("processPhysical", Map.of("status", "pending-shipping")).then(exit()),
                set("handleUnknown", Map.of("error", "invalid-type"))
            )
            .build();
    }
}
----

[#interation]
== 9. Iteration (`for` loops)
Execute logic over arrays of data using the `forTask`.

[source,java]
----
@ApplicationScoped
public class ForWorkflow extends Flow {
    @Override
    public Workflow descriptor() {
        return FuncWorkflowBuilder.workflow("for-example")
            .tasks(
                forTask("checkup")
                    .each("pet").in(".pets")
                    .doTasks(
                        listen("waitForCheckup", to().one(event("pet.checkup.completed")))
                    )
            )
            .build();
    }
}
----

[#parallel]
== 10. Parallel Execution (`fork`)
Execute independent branches simultaneously to reduce total execution time.

[source,java]
----
@ApplicationScoped
public class ForkWorkflow extends Flow {
    @Override
    public Workflow descriptor() {
        return FuncWorkflowBuilder.workflow("fork-example")
            .tasks(
                fork("raiseAlarm")
                    .branch(call("callNurse").http(Method.PUT, "https://hospital.com/nurse"))
                    .branch(call("callDoctor").http(Method.PUT, "https://hospital.com/doctor"))
            )
            .build();
    }
}
----

[#error_handling]
== 12. Error Handling (`try` / `catch`)
Define compensatory logic for business-level errors using `tryTask`.

[source,java]
----
@ApplicationScoped
public class TryCatchWorkflow extends Flow {
    @Override
    public Workflow descriptor() {
        return FuncWorkflowBuilder.workflow("try-catch")
            .tasks(
                tryTask("tryGetPet")
                    .doTasks(
                        call("getPet").http(Method.GET, "https://api.com/pet/{id}")
                    )
                    .catchError(errors -> errors
                        .withType("communication-error")
                        .doTasks(
                            emitJson("notifySupport", "pet.notfound.v1")
                        )
                    )
            )
            .build();
    }
}
----

[#raising_error]
== 13. Raising Explicit Errors
Halt execution and throw specific errors using the `raise` task.

[source,java]
----
@ApplicationScoped
public class RaiseWorkflow extends Flow {
    @Override
    public Workflow descriptor() {
        return FuncWorkflowBuilder.workflow("raise-error")
            .tasks(
                ifCondition(".age < 18")
                    .then(raise("underage-error")
                        .type("access-forbidden")
                        .status(400)
                        .title("Access Forbidden"))
            )
            .build();
    }
}
