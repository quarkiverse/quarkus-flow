Main Workflow Flow

ORDER#001 Scenario

1. Workflow starts
2. Stock Reservation FAILS (ERR_001, status 409)
throw WorkflowException(ERR_001, 409)

ORDER#002 Scenario

1. Workflow starts
2. Stock Reservation succeeds
3. Payment Processing FAILS (ERR_002, status 503)
4. Compensation: Cancel Stock
5. Workflow ends (compensated)
throw WorkflowException(ERR_002, 503)

ORDER#003 Scenario

1. Workflow starts
2. Stock Reservation succeeds
3. Payment Processing succeeds
4. Shipping FAILS (ERR_003, status 500)
5. Compensation: Cancel Payment → Cancel Stock
6. Workflow ends (compensated)
throw WorkflowException(ERR_003, 500)

Technical Implementation

Workflow Pattern

  • • Try-Catch Blocks: Each step wrapped in error handling
  • • Compensation Logic: Automatic rollback on failures
  • • Error Types: Specific error codes (ERR_001, ERR_002, ERR_003)
  • • Status Codes: HTTP-like status codes (409, 503, 500)

Real-time Updates

  • • WebSocket: Bidirectional communication
  • • Events: started, completed, failed
  • • Broadcasting: All connected clients notified
  • • Auto-reconnect: Resilient connection handling
